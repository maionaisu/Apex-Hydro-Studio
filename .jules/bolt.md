## 2024-05-21 - [Replaced cKDTree with Vectorized O(N) Squared Distance for Single-Point Spatial Queries]
**Learning:** For single-point nearest neighbor queries on large unstructured meshes, building a `scipy.spatial.cKDTree` is an anti-pattern. Its O(N log N) construction overhead makes it significantly slower than calculating the squared distance directly and using `np.argmin` (O(N)), especially when the tree is only queried once per dataset load.
**Action:** Always use vectorized squared distance `(dx**2 + dy**2)` combined with `np.argmin` for single-point spatial lookups. Only apply `np.sqrt` to the minimum value found to avoid computationally expensive O(N) `np.hypot` calculations. Reserve `cKDTree` for scenarios requiring multiple queries on the same pre-built tree or highly complex multi-dimensional nearest neighbor searches.
