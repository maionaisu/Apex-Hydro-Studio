## 2024-05-10 - Accessibility Pattern: Form Layout Buddy Pairing
**Learning:** In PyQt6, `QFormLayout.addRow("String", widget)` automatically creates a label and sets the buddy. However, `QFormLayout.addRow(QLabel, widget)` does NOT auto-set the buddy. The application has a composite form component `FormRow` and uses `QFormLayout.addRow(QLabel, widget)` in numerous files. Screen readers struggle with this lack of explicit buddy binding.
**Action:** Always manually explicitly link composite controls or `QLabel` added to `QFormLayout` with `label.setBuddy(widget)`. I have added an explicit `lbl.setBuddy(input_widget)` in the core `FormRow` component. However, many modules are not using `FormRow` and directly using `QFormLayout.addRow(QLabel, widget)`.
## 2024-05-10 - Screen Reader Accessible Buttons
**Learning:** Screen readers struggle to correctly interpret text containing emojis or unicode icon characters, which are heavily used in PyQt6 `QPushButton` instances for visual appeal (e.g. `◀ Peta Sebelumnya`).
**Action:** Always auto-strip emojis using the precise unicode regex block `[\U0001F300-\U0001FAFF\u25A0-\u25FF\u2700-\u27BF\u2600-\u26FF\u2B00-\u2BFF\u2300-\u23FF\uFE0F]` and set this clean string to `setAccessibleName()`. I've applied this pattern directly in the foundational `ModernButton` class to ensure it cascades across all instances application-wide.
