## 2024-05-24 - Accessibility for Emojis in Buttons
**Learning:** Screen readers often struggle to interpret emojis correctly. When emojis are used for visual context in UI components like `ModernButton` (e.g., `▶ JALANKAN ENGINE`), they can degrade the screen reader experience.
**Action:** When adding emojis to UI elements like buttons in PyQt6, use a specific regex targeting known emoji and UI symbol ranges (e.g., `r'[\U00010000-\U0010ffff\u25A0-\u25FF\u2700-\u27BF\u2600-\u26FF\u2B00-\u2BFF\u2300-\u23FF◀▶➔⏳■✅❌⚠]'`) to safely strip them. Avoid aggressive `[^\x00-\x7F]` regexes as they destroy internationalized text (e.g., accented characters, Cyrillic). Supply this plain text equivalent via `setAccessibleName()`. When toggling states, explicitly handle empty strings (e.g., for icon-only buttons) to prevent the accessible name from getting stuck in an old state. Avoid setting redundant `ToolTip` attributes on standard text buttons to prevent visual UX regression.
