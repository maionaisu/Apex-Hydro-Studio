## 2024-05-05 - Accessible Labels and Emoji Stripping for PyQt6
**Learning:** PyQt6 `QLabel` elements used in forms need to be explicitly linked to their input widgets using `setBuddy()`. Without this, screen readers fail to associate the label text with the input field, creating a confusing experience. Furthermore, PyQt6 screen readers struggle with emojis; aggressive ASCII stripping destroys accented characters, so targeting specific Unicode blocks (e.g., `[\U00010000-\U0010ffff\u25A0-\u25FF\u2700-\u27BF\u2600-\u26FF\u2B00-\u2BFF\u2300-\u23FF]`) is necessary to clean up `setAccessibleName()`.
**Action:** Always link form labels to inputs using `QLabel.setBuddy(input_widget)`. When creating generic components with emojis (like `ModernButton`), dynamically strip emojis using targeted regex for `setAccessibleName()` to preserve international text while ensuring screen reader compatibility.
