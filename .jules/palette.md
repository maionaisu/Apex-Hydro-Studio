## 2025-03-01 - [A11y Emoji Fix]
**Learning:** Screen readers often struggle with emojis, reading the raw unicode names out loud (e.g. \"High voltage sign Test\"). In PyQt6's context, stripping this via the application layer using `setAccessibleName()` provides a better user experience for screen reader users. The emoji block `[\U0001F300-\U0001FAFF\u25A0-\u25FF\u2700-\u27BF\u2600-\u26FF\u2B00-\u2BFF\u2300-\u23FF\uFE0F]` is preferred instead of blindly stripping the astral plane range to avoid removing characters important to CJK ideographs and mathematical symbols.
**Action:** When adding standard or customized UI widgets in the future (like `ModernButton`), implement automated `setAccessibleName` updating based on the text contents with emojis aggressively stripped. Always check `if clean_name:` before applying to prevent clearing empty names completely on icon-only buttons.
