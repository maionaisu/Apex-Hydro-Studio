## 2025-06-01 - Screen Reader Compatibility with Emojis in PyQt6
**Learning:** Screen readers struggle to correctly interpret or pronounce emojis when embedded directly in standard PyQt6 UI components, like `QPushButton`. By default, PyQt6 exposes the visible text as the accessible name, meaning an emoji like "🗺️" in a button (e.g., "🗺️ Coastline") causes the screen reader to read out uninterpretable unicode characters or read it out in a confusing way.
**Action:** Always strip emojis (using targeted `re` Unicode ranges) when applying the `setAccessibleName()` for standard UI elements that contain decorative emojis in their visible text. In `ModernButton`, the `_update_accessibility()` method ensures that `setAccessibleName()` provides a clean, text-only alternative (e.g., "Coastline") while preserving the visual emoji in the original text, and it's hooked to `setText()` to handle dynamic updates.

The precise emoji regex block used is: `r'[\U0001F300-\U0001FAFF\u25A0-\u25FF\u2700-\u27BF\u2600-\u26FF\u2B00-\u2BFF\u2300-\u23FF\uFE0F]'`. This avoids accidentally stripping historical scripts, mathematical alphanumeric symbols, and CJK ideographs which might fall in broader unicode ranges.
