## 2026-05-29 - [Emoji Screen Reader Compatibility & PyQt6 Composite Widgets]
**Learning:** Emojis inside buttons cause screen readers to stumble or mispronounce, breaking the accessibility experience. Stripping emojis using a targeted regex block (e.g., `[\U0001F300-\U0001FAFF\u25A0-\u25FF...]`) and explicitly linking `QLabel` to its companion input field using `QLabel.setBuddy()` significantly improves screen reader navigation and pronunciation in composite custom widgets like `FormRow`.
**Action:** When creating text/icon hybrid buttons, calculate an emoji-free `clean_name` and apply it via `setAccessibleName(clean_name)`. Always use `.setBuddy()` when constructing custom form layouts with discrete `QLabel` instances instead of relying on `QFormLayout.addRow()` which sometimes misses this implicit linking.
