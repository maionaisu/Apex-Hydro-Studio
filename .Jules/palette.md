## 2024-05-30 - Emojis vs Screen Readers in PyQt6
**Learning:** PyQt6's `setAccessibleName()` handles emojis poorly across some screen readers. Buttons like "📂 Load Model NetCDF (.nc)" cause issues. Additionally, custom form layouts like `FormRow` (using `QHBoxLayout` with a `QLabel` and an input widget) do not automatically link the label to the input, breaking screen reader association.
**Action:** Always manually link `QLabel` to its corresponding input widget using `QLabel.setBuddy(input_widget)` in custom composite widgets. For text containing UI emojis, use a precise regex `[\U0001F300-\U0001FAFF\u25A0-\u25FF\u2700-\u27BF\u2600-\u26FF\u2B00-\u2BFF\u2300-\u23FF\uFE0F]` to strip them before calling `setAccessibleName()` so that the screen reader gets a clean text string, being careful not to strip generic Unicode.
